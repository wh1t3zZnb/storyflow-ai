# 技术开发进度文档

## 阶段一：MVP 原型验证 (当前阶段)
**目标**：验证“结构化编辑器”的交互体验，跑通“从文字到表格”的流程。
**周期**：1周

*   **Day 1-2**: 需求分析，文档撰写 (已完成)。
*   **Day 3**: 搭建 Vite + React 项目脚手架，配置基础样式架构 (Dark Mode)。
*   **Day 4**: 开发核心组件 `StoryboardTable` (分镜表格) 和 `ScriptCard` (分镜卡片)。
*   **Day 5**: 实现假数据 (Mock Data) 注入，模拟 AI 生成后的效果展示。
*   **Day 6**: 优化 UI/UX，增加微交互 (Hover 效果，选中状态)，确保“专业感”。
*   **Day 7**: 内部演示，收集反馈。

## 阶段二：AI 文本能力接入
**目标**：让“生成”按钮真正可用，接入大语言模型。
**周期**：2周

*   **Week 2**:
    *   后端服务搭建 (Node.js/Python)。
    *   接入 Gemini/OpenAI API。
    *   **核心攻坚**：Prompt Engineering，确保 AI 稳定输出 JSON 格式。
*   **Week 3**:
    *   实现“一键扩写”功能。
    *   实现“流式输出” (Streaming)，让用户看到剧本逐行生成的过程。

## 阶段三：多媒体生成集成
**目标**：闭环“文生图”、“文生视频”工作流。
**周期**：3-4周

*   **Week 4-5**:
    *   接入图像生成 API (Midjourney/SD)。
    *   开发“图片管理”模块：重绘、变体、下载。
*   **Week 6-7**:
    *   接入视频生成 API (Runway/Pika)。
    *   实现简单的“时间轴”预览功能，将分镜连成动态视频。

## 里程碑 (Milestones)
- [x] **M0**: 文档与设计概念冻结。
- [ ] **M1**: 静态高保真 Demo 完成 (可交互，无后端)。
- [ ] **M2**: AI 文本引擎上线 (可生成文字剧本)。
- [ ] **M3**: 全功能 Beta 版本 (含生图功能)。
