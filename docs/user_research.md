# 用户研究报告：AI 智能分镜脚本工具

## 1. 背景与概述
当前内容创作领域（短视频、小说、漫画）爆发式增长，创作者对于“从灵感到落地”的效率要求极高。然而，现有的工作流严重割裂，导致创作者在“创意生成”与“资产管理”之间消耗大量精力。本报告旨在分析目标用户的核心痛点，并验证“AI 智能分镜脚本工具”的市场价值。

## 2. 目标用户画像 (User Personas)

### 2.1 核心用户群
1.  **短视频编导/自媒体创作者**
    *   **特征**：追求更新频率，需要快速产出脚本，对画面感有要求但绘画能力弱。
    *   **需求**：快速把一个梗/段子变成可拍摄的分镜脚本。
2.  **网文/小说作者**
    *   **特征**：文字功底强，但难以将文字视觉化（用于做封面、插图或漫改）。
    *   **需求**：辅助构建世界观，将文字描述转化为具体的角色和场景图。
3.  **独立动画/漫画创作者**
    *   **特征**：专业度高，但重复性工作多（如画分镜草图）。
    *   **需求**：提高分镜绘制效率，管理大量的资产文件。

## 3. 核心痛点分析 (Pain Points)

| 痛点维度 | 具体表现 | 用户原声 (Voice of Customer) |
| :--- | :--- | :--- |
| **工具割裂** | 对话在 ChatGPT，存图在 Midjourney，排版在 Excel，文件在本地文件夹。 | “我每天要在四个软件之间切来切去，复制粘贴手都酸了。” |
| **资产管理混乱** | 生成的几百张图片堆在下载文件夹里，文件名全是乱码，根本对不上是哪一场戏的。 | “过了一周我想找之前生成的男主角侧脸图，根本找不到了。” |
| **非结构化困境** | AI 给的是一大段文字，转换成能用的拍摄脚本需要人工拆解。 | “AI 写的剧本还要我一行行复制到表格里，还要自己标景别。” |
| **提示词门槛** | 知道想要什么画面，但写不出专业的绘画 Prompt。 | “我脑子里有画面，但 AI 生成出来的总是奇奇怪怪。” |

## 4. 解决方案价值主张 (Value Proposition)

**“从流式对话到结构化生产力”**

1.  **一站式工作台**：
    *   左侧对话（灵感），中间表格（结构化），右侧预览（可视化）。
    *   **价值**：减少 90% 的跨软件切换时间。
2.  **自动结构化**：
    *   AI 自动将自然语言剧本拆解为：【场号】【景别】【画面描述】【台词】【备注】。
    *   **价值**：直接生成可用的工业标准文档。
3.  **资产与剧本强绑定**：
    *   生成的图片直接“挂载”在对应的分镜行上，不再是散落的文件。
    *   **价值**：所见即所得，资产管理零负担。

## 5. 结论
市场急需一款能够**整合 AI 生成能力与专业工作流**的工具。MVP (最小可行性产品) 应优先解决“剧本结构化”和“可视化编辑”的问题，为后续的“一键生图/生视频”打好地基。
